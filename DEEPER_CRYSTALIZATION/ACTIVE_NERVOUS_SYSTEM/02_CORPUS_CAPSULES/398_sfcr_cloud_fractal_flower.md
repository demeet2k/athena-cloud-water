## Six Primitive Bridge Families

The SFCR cross-lens algebra is generated by six primitive bridge families. All other transports are composites, reversals with witness data, or refinements.

### B_SF: Square -> Flower (EXACT)

```
B_SF(x,y,K) = (z, w, K)    where z = x+y (mod 4), w = x-y (mod 4)
```

At higher depth: digitwise relation-word lift. Preserves: KernelID, PayloadID, WitnessBundle, MobiusEq. Square says what is committed; Flower says how that commitment is being traversed.

### B_FC: Flower -> Cloud (EXACT on admissible, ILLEGAL otherwise)

```
B_FC(z,w) = F_{z,w}     admissible iff z === w (mod 2)
```

At higher depth: `C_k(Phi_k) = Product F_{z_t, w_t}` with exact branch count `2^k`. Preserves: KernelID, PayloadID, WitnessBundle, AdmissibilityClass.

### B_CR: Cloud -> Fractal (EXACT or RESIDUAL)

```
B_CR(mu) = sigma_6       if support collapses coherently
         = R(mu)          otherwise (residual packet)
```

Cloud does NOT always collapse to a single seed. Sometimes the lawful output is a residual packet. Preserves: KernelID, WitnessBundle (PayloadID/ReplaySpec only on coherent collapse).

### B_RS: Fractal -> Square (EXACT on lawful replay)

Replay/expansion. The return from compressed seed to committed visible state. Preserves: KernelID, PayloadID, ReplaySpec, MobiusEq. If replay fails, the state is illegal.

### B_SR: Square -> Fractal (EXACT, compressive)

Collapse/germ bridge. Turns "board" into "portable replay object." Preserves: KernelID, PayloadID. Intentionally transforms representation from visible state to seed/replay capsule.

### B_CS: Cloud -> Square (REFINEMENT, NOT a function)

```
B_CS: C_d ~~> S_d   (refinement operator)
```

This is the sharpest correction in the corpus. Cloud is lawful multiplicity; returning to Square requires branch choice or witness refinement. Classes:
- REFINEMENT when witness-guided branch choice is available
- RESIDUAL when support is lawful but unresolved
- ILLEGAL if a fake point is emitted without witness

## Principal Loop Closure

```
S_d --[B_SF]--> F_d --[B_FC]--> C_d --[B_CR]--> R_d --[B_RS]--> S_d
```

When each stage is exact: `B_RS . B_CR . B_FC . B_SF ~ id_{S_d}`. This is the maturity criterion.

## Derived Composites

- **Square -> Cloud**: `B_SC = B_FC . B_SF` (exact on lawful inputs)
- **Flower -> Fractal**: `B_FR = B_CR . B_FC` (exact only on coherent collapse, else residualized)
- **Flower -> Square**: should factor as `F -> C ~~> S` (not a primitive)

## Execution Class Matrix (dimension-invariant)

| Bridge | Class | Condition |
|--------|-------|-----------|
| B_SF   | EXACT | always |
| B_FC   | EXACT / ILLEGAL | admissible / inadmissible |
| B_CR   | EXACT / RESIDUAL | coherent collapse / otherwise |
| B_RS   | EXACT / ILLEGAL | lawful replay / uncertified |
| B_SR   | EXACT (compressive) | always |
| B_CS   | REFINE | always (never point-function) |

The execution class is dimension-invariant across the carrier ladder. What changes with dimension is the carrier packet, not the truth-type of the bridge.
